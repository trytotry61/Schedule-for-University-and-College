Система управления и просмотра расписания учебных заведений


Введение 

В настоящее время многие учебные заведения публикуют расписание для учащихся и для прреподавателей в формате файлов pdf/excel
или же в форме огроммной базы вкладокгде содержаться все группы всех факультетов,что неудобно,как для студента,так и для преподавателя.
Данный проект предназначен для облегчения ознакомления с расписаннием для студентов и педагогов.
Мой проект решает большинство из этих проблем
Цели 
1.Повышение удобства доступа к расписанию
2.Снижение нагруки на сотрудников
3.Повышение  уровня цифровизации в сфере образования
Ценность для учебного заведения
Удобство для стдентов
Для студентов:
Быстрый доступ к расписанию с телефона
Поддержка офлайн-режима
Чёткое разделение чётных и нечётных недель
Визуальная ориентация по текущему времени
Для администрации:
Централизованное управление расписанием
Быстрые и легкие массовые операции (копирование недель, замены преподавателей)
История изменений (аудит)
Для  тенического отдела:
Чистая архитектура
Современные технологии
Простота сопровождения и масштабирования
Возможность интеграции с существующими системами
Основные возможности
Для студента:Вход по индивидуальному логину и паролю,
Просмотр только своего расписания,Работа в офлайн режиме
Для администрации:Управление группами,массовые изменения,редактирование расписания,история правок(для быстрого исправления ошибок)
Безопасность проекта:
Индивидуалные учетные записи(логин ипароль можно будет распределить с помощью тьюторов и кураторов лично каждому студенту.)
Защита от несанкционированного входа 
Контрольизменений 
Масштабируемость:
Ближайшие возможности:внедрение системы уведомлений об изменениях.Добавление не только учебных занятий но и внеучебной деятельности "Студеньческая весна" форумы конференции".
Сценарии внедрения:
Тестирования на одном факультете или институте,а заетм постепенное расширение на весь университет с целью избежания множества ошибок,есди программа,другне справится с 
большим количеством студентов.
Вывод:
Мой проект полностю написан и работает без критических ошибок,ему осталось только пройти проверку с большим количесвтом пользователей,но уже сейчас он может быть интегрирован на уровне группы или института,
а затем расшерен на всеь университет.
